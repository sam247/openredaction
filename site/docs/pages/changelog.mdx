# Changelog

All notable changes to OpenRedaction will be documented in this file.

**Note:** OpenRedaction is offered as an open-source library for self-hosted use. A previously offered hosted API and Pro tier are no longer available. For enterprise support and custom deployments, see [Enterprise support](https://openredaction.com/pricing).

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.1.1] - 2026-03-19

### Note
Published as **1.1.1** on npm because **1.1.0** was already present on the registry. Git tag **v1.1.0** matches this codebase; use **1.1.1** on npm for the current build.

### Added
- `repository`, `bugs`, and `homepage` fields in `package.json` for npm metadata

## [1.1.0] - 2026-03-18

### Fixed
- API server routing uses request method and pathname correctly
- Worker bundle included in build for `WorkerPool`
- Streaming file processing compatible with Node `Readable` streams without async iteration
- Request body size limits enforced on the API server

### Changed
- `enableFalsePositiveFilter` defaults to `on`
- Removed hosted AI assist options and exports from the core library
- **`APIServer` / `PrometheusServer` moved to `openredaction/server`** so the main `openredaction` entry does not bundle `node:http` (supply-chain / static analysis friendly). Replace `import { APIServer } from 'openredaction'` with `import { APIServer } from 'openredaction/server'` (same for `createAPIServer`, `PrometheusServer`, `createPrometheusServer`, and related types).

### Added
- NER-only entities merged into the hybrid pipeline when NER is enabled
- Validators: SWIFT/BIC format, Canadian SIN (Luhn), Australian TFN (weighted checksum)
- Presets: `pci-dss`, `soc2`
- Tests for Express middleware, JSON/CSV document processors, and `LocalLearningStore`

## [1.0.0] - 2025-01-15

### Added
- Initial public release
- Regex-first PII detection with 500+ patterns
- Optional AI assist for enhanced detection
- npm library for Node.js
- Hosted API with AI assist
- Support for GDPR, HIPAA, and CCPA presets
- Multiple redaction modes (placeholder, mask, remove)
- Self-hosting capabilities
- Comprehensive documentation

### Hosted API (deprecated)
The following applied to a previously offered hosted API, which is no longer available. Use the library and self-host for production.
- `POST /v1/ai-detect` endpoint (deprecated)
- API key authentication (deprecated)
- Rate limiting and usage tracking (deprecated)

### Library Features
- `redact()` function for simple text redaction
- `OpenRedaction` class for advanced usage
- TypeScript type definitions
- Browser and Node.js support

### Detection Types
- Email addresses
- Phone numbers
- Person names (with AI assist)
- Social Security Numbers
- Credit card numbers
- IP addresses
- Dates of birth
- Physical addresses
- Passport numbers
- Driver's license numbers
- And many more

## Versioning Policy

OpenRedaction follows [Semantic Versioning](https://semver.org/):

- **MAJOR** version for incompatible API changes
- **MINOR** version for backwards-compatible functionality additions
- **PATCH** version for backwards-compatible bug fixes

### Library Versioning

The npm library follows Semantic Versioning. Backward compatibility is maintained within a major version; breaking changes require a new major version.

## Migration Guides

### Library Updates

No breaking changes in the npm library API. All existing code should continue to work.
