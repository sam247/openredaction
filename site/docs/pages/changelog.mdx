# Changelog

All notable changes to OpenRedaction will be documented in this file.

**Note:** OpenRedaction is offered as an open-source library for self-hosted use. A previously offered hosted API and Pro tier are no longer available. For enterprise support and custom deployments, see [Enterprise support](https://openredaction.com/pricing).

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.1.2] - 2026-03-19

**Install:** `npm install openredaction@1.1.2` — this is the current **npm**, **git tag (`v1.1.2`)**, **GitHub Release**, and **site** headline version.

### Fixed
- API server uses correct HTTP method and pathname for routing
- Worker bundle emitted for `WorkerPool`
- Streaming file processing compatible with Node `Readable` streams when async iteration is unavailable
- Request body size limits enforced on the bundled API server

### Changed
- `enableFalsePositiveFilter` defaults to `on`
- Removed hosted AI assist from the library public API and documentation
- **`APIServer`, `createAPIServer`, `PrometheusServer`, and `createPrometheusServer`** are exported from **`openredaction/server`** only, so the main package entry does not reference `node:http`

### Added
- NER-only entities merged into the hybrid pipeline when NER is enabled
- Validators: SWIFT/BIC, Canadian SIN (Luhn), Australian TFN (weighted checksum)
- Presets: `pci-dss`, `soc2`
- Tests: Express middleware, JSON/CSV processors, `LocalLearningStore`, server entry
- CI: GitHub Release created when a `v*.*.*` tag successfully publishes to npm
- `repository`, `bugs`, and `homepage` in published `package.json`

## [1.0.0] - 2025-01-15

### Added
- Initial public release
- Regex-first PII detection with 500+ patterns
- Optional AI assist for enhanced detection
- npm library for Node.js
- Hosted API with AI assist
- Support for GDPR, HIPAA, and CCPA presets
- Multiple redaction modes (placeholder, mask, remove)
- Self-hosting capabilities
- Comprehensive documentation

### Hosted API (deprecated)
The following applied to a previously offered hosted API, which is no longer available. Use the library and self-host for production.
- `POST /v1/ai-detect` endpoint (deprecated)
- API key authentication (deprecated)
- Rate limiting and usage tracking (deprecated)

### Library Features
- `redact()` function for simple text redaction
- `OpenRedaction` class for advanced usage
- TypeScript type definitions
- Browser and Node.js support

### Detection Types
- Email addresses
- Phone numbers
- Person names (with AI assist)
- Social Security Numbers
- Credit card numbers
- IP addresses
- Dates of birth
- Physical addresses
- Passport numbers
- Driver's license numbers
- And many more

## Versioning Policy

OpenRedaction follows [Semantic Versioning](https://semver.org/):

- **MAJOR** version for incompatible API changes
- **MINOR** version for backwards-compatible functionality additions
- **PATCH** version for backwards-compatible bug fixes

### Library Versioning

The npm library follows Semantic Versioning. Backward compatibility is maintained within a major version; breaking changes require a new major version.

## Migration Guides

### 1.1.x — HTTP server imports

If you used the REST API server or Prometheus HTTP server from the main package entry, switch to the server subpath:

```ts
// Before
import { APIServer, createPrometheusServer } from 'openredaction';

// After
import { APIServer, createPrometheusServer } from 'openredaction/server';
```

### Library updates

Other `openredaction` imports remain backward-compatible within 1.x unless noted in release notes.
